Social Media Project: Dawgr

A React Native Application

Project Background

Since the early 2000's, social media usage has been growing exponentially and its growth is still unihibited almost 20 years on. While many social media sites serve as sources for news and information, they are capable of much more. Indeed, the world has discovered their ability to allow people to interact amongst themselves in addition to the provider-customer commercial relationship. The barrier to the simple exchange of ideas, photos and almost any content has been removed as structured websites and blogs take a diminished role. As the use of social media platforms expand, they tend to develop more unique audiences.

DAWGr is an example of a platform directed toward dog owners and people who share a love of pets.

Requirements

The DAWGr project presents a mobile user interface for the following areas:

  • Main view: Landing page that displays three sections of dog categories: top rated, newest posts and bottom rated dogs.
  • Full List view: When a user clicks the "See more" link at the bottom of each category, the view will update to this page of up to 100 image thumbnails.
  • Detail view: When a user clicks on a thumbnail from any view, the screen will update to this larger image display with icons to review or save the dog as a favorite.
  • Profile view: A form-driven page for a user to upload pictures of their dog along with captions and other information.
  • Contest view: Shows the winner from the last competition plus thumbnails of the latest contest entries.
  • Favorites: A display of the user's selected favorites (from clicking heart icon in image view).
  • Settings: User and application settings.

The design of the application was rooted in the use of Expo to achieve a solution for both iOS and Android using React Native. Several React Native component libraries accelerated the implmentation:

Section Grid: The react-native-super-grid library was used in the Main view:

  • Defined three sections for each data type.
  • Coded my own rendering callback functions to display thumbnails within the grid sections.
  • Created header and footer callback functions in each grid. The footer contains the navigation to the Full view.

Carousels: To implement a carousel display for sponsored advertisements, I utilized react-native-slick. The Slick component allowed me to rotate Cards for each sponsor object.

Image Picker: I explored several different image selection librarires before settling on ImagePicker from "expo-image-picker". This library provides support for both iOS and Android with near-identical functionality to select and apply aspect ratio cropping to an image. In addition to ImagePicker, the following related libraries were needed:

  • Permissions from 'expo-permissions': Prompts the user for access to media library and camera.
  • ImageManipulator from 'expo-image-manipulator': Enables sizing and file format upon save.
  • MediaLibrary from 'expo-media-library': Enables writing to the media library on the device.

Actions & Outcomes

While the mentioned React Native libraries greatly aided the development of DAWGr, a great deal of care was needed to achieve the navigation scheme that I intended. The use of top tab navigators that separate image viewing functionality from image uploading was fairly straightforward with stack navigation working underneath. However, I also wanted a control bar above the tabbed navigation to offer global functions (favorites, contests and settings at the current time plus room for any future features). The key was to implement an additional stacked navigator of type "modal" and allow the icon-directed screens to also be the modal type.

I used Redux to manage the application state for all data and images, with the exception of logo-level images. This allows for centralized real-time updating of displayed information: image collections, ratings, comments, advertising, etc. The NodeJS/Express/MongoDB backend server enabled a clean separation of the front-end logic from the programmatic data.

  • React Native (and Expo) allowed for a single JS-driven code base for multiple native devices.
  • Reusable components streamlined the code and reduced total lines.
  • Redux management of state provided a centralized interface to the backend.

Conclusions

While React Native indeed delivers a single code stack, there were a few platform-specific issues that had to be addressed. For example, the iPhone X viewport needed an additional reserved area for the "notch" on the device's screen. Android also has an issue of the keybord pop-up covering up portions of a form that are receiving input (user loses visibility of what's being entered). I discovered a remedy in the library, "react-native-keyboard-aware-scroll-view", which auto-scrolls the screen for this scenario.

The DAWGr project is a fun experiment that explores the idea of a focused audience that is free to create and share content while also demonstrating areas of commercial advertising and sponsorship.
