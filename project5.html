Mobile Tourism Project: Olympia Top 10

A React Native Application

Project Background

This project is the extension of my Olympia Top 10 React web application to a React Native mobile application for iOS and Android. Unlike my other React Native projects that were conceived as mobile-first, I wanted to explore what might be different or what could be improved by transforming an existing React application (that already had responsive web design from ReactStrap) to React Native.

Requirements

The Mobile Olympia Top 10 project presents a mobile user interface for the following areas:

  • Main view: Landing page that presents site drawer navigation, branding and promotional area.
  • Guide view: When a user clicks the navigation to Attractions, Restaurants or Events, the Guide component is called to render the top ten items in each category.
  • Detail view: When a user clicks on an item from a top 10 list, a larger image will be displayed with more detailed information about the item.
  • Gallery view: When a user clicks on a "more" button (when presented) the screen will update to this "masonry" gallery view that's geared for small viewports.
  • Resources view: Instead of a carousel of cards (like in the React app), the React Native application will offer the Resources as another navigation point within the drawer navigator.
  • Contest entry: A modal that appears when "enter sweepstakes" is selected from the Main view.

The mission of the React Navtive app was to streamline the navigation and presentation for mobile viewports in a way that the responsive React app can't. For example, the mobile application makes use of drawer navigation that responds to screen swipes and it gets out of the way after selections are made. I also wanted to preserve many of the features and behaviors that were in the React app, such as: animation and galleries. I chose libraries that enabled easier implmentation of these features:

Animation: I used Animatable from the library "react-native-animatable" to perform several effects: a "tada" splash of the the site's logo and a "fade-in" animation of the site's tag line.

Guide: I utilized react-native's "FlatList" component to render react-native-elements "Tile" components for each of the top 10 items.

Details: When one of the top 10 items is clicked, I utilize a HighLightInfo component to render a detailed view. This component constructs a card and formats three types of links if the data is present: a link to a Google Maps view of the address, a link to the external website and a navigation link to an image gallery view of more images.

Image Gallery: If additional images are present in the top 10 data object, a button will be presented that navigates to the ImageGallery component. Here, I use MasonryList from the "react-native-masonry-list" library to generate a "masonry" style gallery for smaller viewports.

Carousels: To implement a carousel display for sponsored advertisements, I utilized react-native-slick. The Slick component allowed me to rotate Cards for each sponsor object (local Olympia businesses).

Actions & Outcomes

React enabled great opportunities to create reusable components for this project. In particular, the code for the three top ten lists became one... only with different data being passed to the component.

React also enabled a clean separation of the front-end logic from the programmatic data. Virtually all data is stored in the Redux Store for the application. The foundational work in the React web application paved the way for a much easier path for a React Native version which is also detailed in the projects page.

I also made a few discoveries along the way. While the masonry gallery for React Native had a nice algorithm for making a column-based collage of images, the viewport limited the average image size much more than its React counterpart. As a result, I implemented my own full-screen modal to display any image that gets clicked from the masonry gallery. I also discovered that I couldn't easily use all of the CSS tricks that I implemented in the React app. In particular, I dropped the purple image overlay in the corner that displays a ranking number. Instead, I used Tiles to show a list of images with a purple arrow to click for more details.

  • React Native (and Expo) allowed for a single JS-driven code base for multiple native devices.
  • Reusable React Native components were quickly derived from the completed React web application.
  • Redux management of state provided a centralized interface to the program data.

Conclusions

The Mobile Olympia Top Ten project is a useful experiment that allowed me to transform an already-responsive web application into a pure native application for mobile devices. The palette of options are rich for both React and React Native, and I was able to find comparable component libraries to get the features I wanted. In the end, I was able to refocus the user experience for the smaller viewports by taking advantage of the drawer navigation that's more unique to devices and by moving the tourist resources from a carousel into it's own list view. Some things did translate easily -- the data and state are virtually identical between the two front-end platforms, and the general structure of class components remained the same.
