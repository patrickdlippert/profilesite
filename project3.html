Tourism Project: Olympia Top 10

A React Application

Project Background

For over three decades, the tourism industry has been relying on the Internet as a way to reduce their traditional marketing strategies of mass advertising. While large web portals like Hotels.com, Expedia and Travelocity have had success in a global marketplace, smaller cities may still find a need to directly market to a targeted audience. Indeed, who would know better about the ins and outs of a particular area than the locals?

The Olympia Top 10 is an example of a platform directed toward people visiting or recently moving to Olympia Washington. The site also serves as a resource for locals who want to keep tabs on what's new, and particularly what people judge to be the best.

Requirements

The Olympia Top 10 project presents a responsive user interface for the following areas:

  • Main view: Landing page that presents site navigation, branding, promotional area and a carousel of resources.
  • Guide view: When a user clicks the navigation to Attractions, Restaurants or Events, the Guide component is called to render the top ten items in each category.
  • Detail view: When a user clicks on a "more" button (when presented) the screen will update to this "masonry" gallery view.
  • Profile view: A form-driven page for a user to upload pictures of their dog along with captions and other information.
  • Contest entry: A modal that appears when "enter sweepstakes" is selected from the Main view.

The core design was that all of the page views should be unified by matching header and footer branding and navigation. This was the starting point for converting the design from wireframe into code. The wireframes provided basic layout of graphics, text and form details, but I spent some time choosing:

Footer: The footer shows the expected simple navigation links for when the page is fully scrolled, but it also includes a Washington State graphic that provides a point of reference for the city of Olympia's location relative to Seattle.

Guide: I utilized the mdbreact library (Material Design Boostrap) to use a zoom-in effect on each item in the top ten list while also alternating images & text down the page for visual interest. If addition images are included in the provided data objects, a button and route link will be generated for a "masonry" gallery view.

Carousels: To implement a carousel display the full pages, I utilized the Slider component from react-slick. Slider allowed me to construct my own CardCarousel component where I rotate cards with images for the resource links on the Main view. I reused the CardCarousel component for local sponsors/advertisers at the bottom of the three category views.

Actions & Outcomes

React enabled great opportunities to create reusable components for this project. In particular, the code for the three top ten lists became one... only with different data being passed to the component.

React also enabled a clean separation of the front-end logic from the programmatic data. Virtually all data is stored in the Redux Store for the application. The foundational work in the React web application paved the way for a much easier path for a React Native version which is also detailed in the projects page.

  • React transformed a multi-page concept into a single page app.
  • Reusable components streamlined the code and reduced total lines.
  • Redux management of state provided a centralized interface to the data.

Conclusions

The Olympia Top Ten project is a useful experiment that explores the idea of a localized travel application that offers helpful guide content while also demonstrating areas of commercial advertising and sponsorship. It utilizes a good amount of animation effects from MDB (Material Design Bootstrap) to keep a lively presentation on all views.
