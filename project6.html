JavaScript Matching Game: Minions

A Vanilla JavaScript Application

Project Background

This is a game that uses JavaScript to directly interact with the DOM to present everyone's favorite workers. It's an old-school type of game where you compare the left side of the playing field to the right side and click on the additional item. If you guess correctly, a team of three minions will cheer you on with a variety of happy minion sounds. However, if you click on the wrong minion, you'll see a pair of sad minions who will lament in your failure.

Requirements

I wanted to create a matching game that had a few more features than a simple comparision of one image. I set out to add the following features:

  • Play Modes: I created four different play modes:
    • Easy (adds two more minions to each successful level)
    • Normal (adds five minions to each successful level)
    • Difficult (adds 8 minions to each successful level)
    • Extreme (adds 12 minions to each successful level and also randomizes a variety of minion types)
  • Sounds: When a player successfully moves to the next level, I randomly play one of 15 minion mp3 audio files.
  • Images: When a player advances, I change the type of minion that gets displayed (in all modes except Extreme, which randomly uses all minion types on every level).
  • Animation: When a player advances, I animate a group of three minions across the screen. The animation alternates left-to-right and right-to-left on each level. When a player guesses incorrectly, I animate a "Game over' modal from the bottom of the screen along with a "Try again?" button to reset the game.

Actions & Outcomes

The JavaScript uses several fundamental methods to manipulate the DOM: appendChild, removeChild, cloneNode, addEventListener & removeEventListener (for clicking on minions). It was a good experiment for using the .play() method to trigger sounds and also using setInterval to trigger position changes to animate images (and clearInterval to stop the animation as well). I also got to opportunity to use the math library to generate random sounds and images to keep the game more interesting for the player.

Conclusions

This was a great experiment to see the power of JavaScript when everything gets stripped down to the basics. Indeed, the HTML for this project is extremely small at 68 lines in the index.html file and about 150 lines of CSS to style it. The HTML code basically defines the playing field divs and displays the fixed logo header and the "Game Over" modal. Almost everything else in handled in a single minions.js JavaScript file.
